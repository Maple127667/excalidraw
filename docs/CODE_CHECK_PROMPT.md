# 🎯 检查目标

* 输出**改进建议**

---

# 🧱 一、项目结构检查（Architecture）

## 1.1 目录结构

检查项：

* [ ] 是否存在清晰的目录分层（app / server / utils / uploads）
* [ ] 前后端职责是否分离明确
* [ ] 是否存在混乱或重复目录
* [ ] 是否存在“临时文件/测试代码未清理”

重点关注：

* `/app`
* `/server`
* `/uploads`
* `/public`

输出：

* 问题描述
* 风险等级（低 / 中 / 高）
* 优化建议

---

## 1.2 模块边界

检查项：

* [ ] API 层是否与业务逻辑解耦
* [ ] 是否存在跨层直接调用（例如页面直接操作 DB）
* [ ] 是否有重复逻辑

---

# ⚙️ 二、后端稳定性（Server / API）

## 2.1 API 设计

检查：

* [ ] REST 结构是否规范
* [ ] 路由是否统一（/api/admin/* /api/public/*）
* [ ] 是否存在重复 API
* [ ] HTTP 方法是否正确（GET/POST/PUT/DELETE）

---

## 2.2 错误处理

检查：

* [ ] 是否有 try/catch
* [ ] 是否返回统一错误格式
* [ ] 是否存在未捕获异常
* [ ] 是否暴露敏感错误信息

---

## 2.3 输入校验

检查：

* [ ] 是否校验请求参数
* [ ] 是否防止非法输入
* [ ] 文件上传是否校验类型/大小

---

## 2.4 文件上传系统

检查：

* [ ] 是否使用 `uploads/` 而不是 `public/`
* [ ] 文件命名是否安全（uuid）
* [ ] 是否存在路径注入风险
* [ ] 是否支持并发上传
* [ ] 是否有文件引用关系管理

---

## 2.5 并发与竞态

检查：

* [ ] 是否存在文件覆盖风险
* [ ] 是否存在 race condition
* [ ] 是否存在重复写入

---

# 🧠 三、Markdown 与内容系统

## 3.1 Markdown 渲染

检查：

* [ ] 是否使用安全渲染（防 XSS）
* [ ] 是否允许 raw HTML（html: true）
* [ ] 是否做了 HTML escape

---

## 3.2 图片处理

检查：

* [ ] 图片路径是否统一（/media/...）
* [ ] 是否存在路径不一致
* [ ] 是否存在失效图片
* [ ] 是否存在硬编码路径

---

## 3.3 内容存储

检查：

* [ ] Markdown 是否存数据库
* [ ] 是否存在内容重复
* [ ] 是否存在无法追踪来源的文件

---

# 🎨 四、前端稳定性（Nuxt3）

## 4.1 页面结构

检查：

* [ ] 页面是否过于庞大
* [ ] 是否拆分组件
* [ ] 是否存在重复组件

---

## 4.2 数据请求

检查：

* [ ] useFetch 是否正确使用 key
* [ ] 是否存在重复请求
* [ ] 是否存在未处理 loading/error

---

## 4.3 路由系统

检查：

* [ ] 是否存在路由冲突
* [ ] 是否误拦截静态资源（如 /images/*）
* [ ] 是否存在 fallback 问题

---

## 4.4 v-html 使用

检查：

* [ ] 是否存在 XSS 风险
* [ ] 是否对内容来源可信
* [ ] 是否有 sanitize

---

# 🔐 五、安全性（Security）

## 5.1 鉴权

检查：

* [ ] 是否存在管理员认证
* [ ] 是否使用 HttpOnly Cookie / Token
* [ ] API 是否保护（/api/admin/*）

---

## 5.2 权限控制

检查：

* [ ] 是否存在未授权访问
* [ ] 是否能绕过权限

---

## 5.3 文件安全

检查：

* [ ] 是否允许上传恶意文件
* [ ] 是否限制 MIME 类型
* [ ] 是否防止路径遍历（../）

---

# 🚀 六、性能（Performance）

## 6.1 前端性能

检查：

* [ ] 是否有大体积图片未压缩
* [ ] 是否懒加载图片
* [ ] 是否有重复渲染

---

## 6.2 后端性能

检查：

* [ ] API 是否有阻塞操作
* [ ] 是否存在同步 IO
* [ ] 是否有缓存机制

---

# 📦 七、部署与环境

## 7.1 环境一致性

检查：

* [ ] dev/prod 是否行为一致
* [ ] 是否依赖本地文件
* [ ] 是否存在路径问题

---

## 7.2 文件持久化

检查：

* [ ] uploads 是否会丢失
* [ ] 是否适配 Docker
* [ ] 是否支持扩展为对象存储

---

# 🧪 八、日志与可观测性

检查：

* [ ] 是否有日志输出
* [ ] 是否记录错误
* [ ] 是否可追踪请求
